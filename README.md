# ⚓ Ahoy — море общения

Социальная сеть в духе X (Twitter) с морским характером. Регистрация по email,
оформление профиля с фотографией и описанием, живая лента с постами,
изображениями и «якорями» вместо лайков.

## Возможности

- **Регистрация и вход по email** — сессии на сервере, пароли хешируются
- **Онбординг профиля** — загрузка аватара и описание «о себе» сразу после регистрации
- **Общая лента** — посты, якоря (лайки) и комментарии видят все пользователи
  с любых устройств; лента автообновляется каждые 15 секунд
- **Профили** — страницы `/u/ник`, редактирование имени, ника, био и фото
- **Редактирование постов** — с пометкой «изм.»
- **Галочка верификации** — для избранных ников
- **Анимации повсюду** — framer-motion, покачивание постов «на волнах»
- **Стеклянный интерфейс** поверх живого фона морской волны

## Запуск локально

```bash
npm install
npm run dev
```

Без настройки базы данные хранятся в памяти процесса — этого достаточно
для разработки, но всё сбрасывается при перезапуске.

## Деплой на Vercel (с общей базой)

Чтобы посты и лайки видели все пользователи с разных устройств, подключите
бесплатную базу Upstash Redis (256 МБ и 500 тыс. команд в месяц — карта не нужна).

**Способ 1 — напрямую через Upstash (рекомендуется, без карты):**

1. Зарегистрируйтесь на [console.upstash.com](https://console.upstash.com)
   (вход через GitHub или Google, бесплатно).
2. **Create Database** → тип Redis, тариф **Free**, регион — ближайший к
   региону функций Vercel (по умолчанию `us-east-1` / N. Virginia).
3. На странице базы найдите раздел **REST API** и скопируйте значения
   `UPSTASH_REDIS_REST_URL` и `UPSTASH_REDIS_REST_TOKEN`.
4. В Vercel: проект → **Settings** → **Environment Variables** → добавьте обе
   переменные (для всех окружений) → **Save**.
5. **Deployments** → меню «⋯» у последнего деплоя → **Redeploy**.

**Способ 2 — через Vercel Marketplace:** Storage → Create Database → Upstash →
Upstash for Redis (Free) → Connect to Project → Redeploy. Переменные добавятся
сами (`KV_REST_API_URL` / `KV_REST_API_TOKEN` — тоже поддерживаются).

Готово — теперь база общая для всех посетителей сайта, а жёлтый баннер
в ленте исчезнет.

> ⚠️ Без подключённой базы на Vercel данные живут в памяти serverless-функции
> и пропадают между запросами — база обязательна для продакшена.

## Как это устроено

Клиент общается с API (`src/lib/api.ts` → `src/app/api/*`), токен сессии
хранится в localStorage. Серверный слой (`src/lib/db.ts`) пишет пользователей,
посты и сессии в Upstash Redis, а без него — во внутреннюю память (для dev).
Аватары и картинки постов сжимаются на канвасе и хранятся как data URL.
